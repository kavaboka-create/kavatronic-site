**Проект:** kavatronic.tech  
**Версия:** 2.0  
**Статус:** утверждено к внедрению (заменяет v1.0 в части структуры главной и IA первого уровня)  
**Дата:** 20.05.2026

**Отношение к v1.0:** архивный документ `TZ_Kavatronic_5_lineek_v1.0_FINAL_ARCHIVED.md` остаётся справочником по позиционированию, каноническим названиям линеек, B2B-тону и техническим ограничениям (в т.ч. реализованному P0). **v2.0 — канон** для структуры главной, hero, карты переходов (product layer / technical layer) и этапности P0/P1.

---

## 1. Цель проекта

### Проблема

Главная страница после рефрейма v1.0 воспринимается как **длинный лендинг**: дублирует каталог (блок 3 KVT), тянет статьи и большую форму, перегружает первый экран текстом. Глубина по линейкам и инженерный контекст должны жить на **отдельных страницах направлений**, а не на главной.

### Решение

Сделать главную **короткой B2B-входной страницей в платформу** из **4 контентных секций** (+ header/footer). Глубина — на **product hubs** (`/products/*`) и **technical pages** (`/platform/*`).

### Целевое позиционирование (без изменений)

**Kavatronic — промышленные компоненты для построения управляемых мехатронных узлов и мобильных систем: исполнение, измерение движения, вычисление, питание и восприятие среды.**

Тон: B2B, integrator/OEM, инженерный, без retail-витрины. Сервоприводы KVT — **якорный коммерческий продукт**, но главная не является каталогом SKU.

## 3. Архитектура навигации первого уровня

На главной действуют **два независимых слоя входа**. Их нельзя смешивать в одну ссылку.

### 3.1. Product layer (hero renders)

**Назначение:** коммерческий / продуктовый вход в линейку.

| Рендер в hero | Целевой URL |
|---------------|-------------|
| Servos / Actuators | `/products/servos/` |
| IMU | `/products/imu/` |
| Master Controller | `/products/controller/` |
| Smart PDU | `/products/pdu/` |
| 2D/3D LiDAR / Depth Camera | `/products/lidar/` |

**IMU (обязательное правило):** переход с hero **не** ведёт на одну SKU-страницу (`products/imu118.html` или `products/imu488.html`). Целевой узел — **IMU hub** `/products/imu/`, объединяющий оба продукта: IMU118 и IMU488. Существующие product URLs сохраняются и доступны с hub.

### 3.2. Technical layer (`#platform`)

**Назначение:** инженерный / архитектурный вход — роль компонента в мехатронном узле.

| Роль / кнопка в `#platform` | Целевой URL |
|-----------------------------|-------------|
| Servos / Actuators | `/platform/servo/` |
| IMU | `/platform/imu/` |
| Master Controller | `/platform/controller/` |
| Smart PDU | `/platform/pdu/` |
| 2D/3D LiDAR / Depth Camera | `/platform/lidar/` |

### 3.3. Содержание technical pages (`/platform/*`)

Каждая страница направления — **статическая HTML-страница** с единым шаблоном:

| Блок | Содержание |
|------|------------|
| Роль в узле | Зачем компонент в типовой мехатронной системе |
| Назначение | Для чего нужен, границы применимости |
| Применения | 3–5 инженерных сценариев (без mil/defense) |
| Интерфейсы / интеграция | Шины, данные, связь с другими ролями узла |
| Материалы | Ссылки на **существующие** статьи `/articles/` по теме |
| Переход в каталог | CTA на соответствующий **product hub** (`/products/.../`) |

Technical page **не** заменяет product hub и **не** дублирует полный каталог SKU.

#### 3.3.1. `/platform/servo/` — обязательный educational-block (P1)

На странице **Servos / Actuators** в technical layer **обязательно** размещается блок, наследующий смысл legacy `#how-servo-works` / «Анатомия стальной мышцы» (ранее на главной, снят в P0).

| Элемент | Требование |
|---------|------------|
| Назначение | Краткое объяснение устройства smart-сервопривода для инженерного входа (technical layer), без каталога SKU |
| Заголовок | «Анатомия стальной мышцы» (или эквивалент в том же смысле) |
| Структура | Краткий ввод (1–2 предложения) + **список 5–7 тезисов** по узлам (корпус/вал, мотор, редуктор, энкодер, плата, драйвер, питание/шина) |
| Полный разбор | Ссылка «Читать полный инженерный разбор →» на **`articles/article-4.html`** (канонический long-form; не дублировать статью целиком на platform) |
| Расширение деталей | Дополнительные абзацы — **свёрнуты** (`<details>` / always-open) **или** через локальный expand; см. §3.5 |
| Якорь (опционально) | `id="how-servo-works"` на секции platform — только для обратной совместимости закладок; **не** на главной |

Блок **не** возвращается на главную. На SKU-страницах (`products/kvt-*.html`) этот educational-block **не обязателен** — там остаётся парное `.product-compare` (§3.4.1).

### 3.4. Product hubs (`/products/*`)

| Hub | Минимальное содержание |
|-----|------------------------|
| `/products/servos/` | Обзор линейки KVT, ссылки на все SKU (`kvt-*.html`), **обязательный** comparison block (§3.4.1) |
| `/products/imu/` | **Обязательно:** IMU118 + IMU488, сравнение по интерфейсу/частоте, единый CTA |
| `/products/controller/` | B2B-описание, статус «по запросу», форма/контакт |
| `/products/pdu/` | Аналогично |
| `/products/lidar/` | Аналогично |

Существующие product URLs (`products/kvt-80r.html`, `products/imu118.html` и т.д.) **не ломать**; hub добавляет слой навигации, не заменяет страницы моделей без 301 (миграция URL — P2).

#### 3.4.1. `/products/servos/` — обязательный comparison block (P1)

На hub сервоприводов **обязательно** размещается сводное сравнение линейки KVT с аналогами Dynamixel / MX (наследие legacy `#comparison` с главной).

| Элемент | Требование |
|---------|------------|
| Роль блока | Сравнение **всей линейки** KVT с популярными аналогами по классу (момент, напряжение, скорость, протокол, тип мотора) — для предварительного подбора |
| Формат | Сводная таблица (классы `.comparison-table` / `.compare-table`, обёртка `.table-scroll`) + disclaimer `.comparison-disclaimer` |
| Строки | Все 6 SKU линейки (KVT-30T, KVT-40D, KVT-40R, KVT-65R, KVT-80R, KVT-260R) с колонкой «Аналог Dynamixel» (MX-28, MX-64AR, XM430, XM540 и т.д. — по утверждённой матрице из pre-P0) |
| Якорь (опционально) | `id="comparison"` на hub — для обратной совместимости; **не** на главной |
| Дублирование | Сводную таблицу **не** размещать на главной и **не** дублировать на каждой SKU-странице |
| SKU-уровень | На `products/kvt-*.html` сохраняются активные парные блоки **`.product-compare`** (детальное сравнение одной модели); hub даёт обзор линейки, SKU — углубление |

### 3.5. Размещение legacy-блоков (канон v2)
